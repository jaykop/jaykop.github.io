## 데이터 처리 모델
* 데이터를 받는 방식을 뜻한다. 

## 동기 처리 모델
* 데이터 요청과 결과가 함께 일어난다
* 이 일련의 과정이 일어나는 동안 다른 작업은 수행되지 않는다
* 설계가 간단하고 직관적이다
* 결과가 도출될 때까지 다른 작업을 수행할 수 없다 (= 블록 상태)

## 비동기 처리 모델
* 데이터 요청과 결과가 별개로 수행된다
* 다른 작업이 동시에, 번갈아가며 수행된다
* 작업이 수행되는 동안 다른 작업을 수행 가능하다 (= 논블록 상태)
* 동기식보다 설계가 복잡하다
